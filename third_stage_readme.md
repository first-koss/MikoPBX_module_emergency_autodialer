# EmergencyAutodialer: итоги третьего этапа

## Что было сделано ранее

Этап 1 превратил шаблон MikoPBX в модуль `EmergencyAutodialer`, который в интерфейсе отображается как `Автооповещение`.

Этап 2 добавил доменную модель данных:

- scope;
- сценарии;
- списки получателей;
- получателей;
- связи сценариев со списками;
- кампании;
- снимки получателей кампании;
- попытки дозвона.

Также был добавлен default scope с кодом `default`.

## Что сделано в третьем этапе

- Версия модуля повышена до `0.3.0`.
- Данные разработчика обновлены:
  - developer: `firstkoss`;
  - support email: `firstkoss@yandex.ru`;

- Обновлены переводы для формы настроек и ошибок.
- `CHANGELOG.md`, `VERSION`, `module.json` и `README.md` обновлены до версии `0.3.0`.

## Что можно проверить после третьего этапа

После установки или обновления модуля в MikoPBX можно проверить:

- модуль отображается как `Автооповещение`;
- в метаданных модуля указан разработчик `firstkoss`;
- открывается `/emergency-autodialer/settings/index`;
- на странице настроек отображается форма default scope;
- можно изменить:
  - название точки входа;
  - внутренний номер запуска;
  - PIN;
  - идентификаторы звуковых файлов;
  - параметры попыток, параллельности и порога прослушивания;
- форма сохраняется через `/emergency-autodialer/settings/save`;
- после перезагрузки страницы сохраненные значения остаются;
- некорректные значения не проходят валидацию;
- страницы кампаний и диагностики продолжают открываться.

## Важное ограничение

Поля `welcome_sound_id` и `invalid_sound_id` пока являются текстовыми идентификаторами. Dropdown выбора существующих звуковых файлов MikoPBX нужно подключить после уточнения core-модели звуковых файлов.
